# Скрипты для автоматизации работы с интернет-магазинами

Проект содержит два скрипта: `seller.py` и `market.py`. Они предназначены для автоматического обновления остатков и цен товаров на маркетплейсах, используя данные от поставщика.

## seller.py

Этот скрипт работает с Озон через API продавца и позволяет:

1. Получать список всех товаров магазина.

2. Обновлять остатки товаров.

3. Обновлять цены товаров.

4. Не перегружает API большим количеством данных за один запрос.

5. Обрабатывает ошибки соединения и таймауты.

### Как работает:

- Скрипт получает актуальные данные о товарах от поставщика.

- Проверяет, какие товары уже есть в системе Озон.

- Формирует структуры данных для обновления остатков и цен.

- Отправляет данные на API Озон.

- Обрабатывает ошибки и таймауты, чтобы не потерять данные.

## market.py

Этот скрипт работает с Яндекс.Маркет через Partner API и предназначен для автоматического обновления остатков и цен товаров магазина.

### Основные возможности:

1. Получение списка всех товаров магазина по кампаниям FBS и DBS.

2. Обновление остатков товаров на складе маркетплейса.

3. Обновление цен товаров в магазине.

4. Работа с файлами поставщика для получения актуальных данных о количестве и цене товаров.

5. Обработка ошибок соединения и таймаутов при работе с API.

### Модели продаж:

FBS (Fulfillment by Seller) — товары хранятся у продавца, Маркет выполняет доставку.

DBS (Dropship by Seller) — товары хранятся у продавца, продавец сам отправляет заказы.

### Как работает:

- Скрипт получает актуальные данные о товарах от поставщика.

- Проверяет товары в кампаниях FBS и DBS на маркетплейсе.

- Создаёт структуры для обновления остатков и цен.

- Отправляет данные на API Яндекс.Маркета.

- Обрабатывает ошибки и таймауты, чтобы данные были корректными.
